---
title: Routing rules
description: Ordered, first-match request routing.
---

Routing rules are an ordered, first-match-wins list evaluated in the proxy [transform pipeline](/docs/concepts/proxy). Each rule can:

- `continue` — pass through unchanged
- `rewrite_model` — map the requested model to another
- `reject` — block the request
- set a per-rule **auth mode**: `require_key` or `passthrough`
- target a configured **direct HTTPS `/v1` upstream**, bypassing llama-swap for that request while keeping proxy logging

Direct targets can reference [encrypted upstream credentials](/docs/access-policy/upstream-credentials). Passthrough rules can preserve the client `Authorization` header for upstream OAuth / API-key validation — this is how [Claude Code passthrough](/docs/clients/claude-code) is configured.
