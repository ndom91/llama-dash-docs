---
title: Upstream credentials
description: An encrypted vault for direct-upstream and MCP-relay secrets.
---

The credential vault stores encrypted bearer credentials for [direct upstream routing](/docs/access-policy/routing-rules) and [MCP relays](/docs/clients/mcp-relays). The admin API never returns secret values; the proxy injects a credential immediately before forwarding and redacts it from request logs.

Bindings can either set an outbound header automatically or replace a namespaced placeholder like `{{llama-dash:credential:<slug>}}`. The vault warns before deleting a credential still referenced by a routing rule or relay.

<Callout type="warn">
  Stored credentials require
  [`CREDENTIAL_ENCRYPTION_KEY`](/docs/reference/environment-variables) to be set to
  a 32+ character value. Credential-bearing rules always require a valid llama-dash
  API key first.
</Callout>
